# 对话记忆规则

## 采用方案三：双机制

### 机制 1：当场提炼
当对话中出现以下内容时，尽量当场提炼并写入长期记忆：
- 明确的长期偏好
- 稳定规则
- 重要决策
- 身份信息
- 反复出现的工作方式

### 机制 2：中午统一复盘
每天中午 12 点后的第一次 heartbeat：
- 复盘当天近期对话中零散但有价值的信息
- 判断哪些值得进入 memory
- 判断哪些值得进入脑 skill
- 判断是否值得形成新 skill 或丰富旧 skill

### 机制 3：结构化沉淀
如果信息已经不只是“记住一句话”，而是形成了：
- 可复用工作流
- 解释框架
- 操作规范
- 长期模块化知识

则优先沉淀到：
- `脑/skill&api/`

---

## 记忆分层

### 1. 当前上下文
适合：本轮任务、临时状态、短期细节

### 2. 长期 memory
适合：用户偏好、规则、重要决策、身份信息

### 3. 脑内 skill
适合：成熟知识、工作流、结构化能力

---

## 不强行记住的内容
- 闲聊
- 一次性噪音
- 很快失效的小细节
- 纯过程性试错记录

---

## 当前执行原则
- 能当场提炼的，当场提炼
- 需要整理的，中午统一复盘
- 能长成 skill 的，不只记一句话，而是做成模块
