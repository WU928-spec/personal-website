# Plan: 将「写文章」页面重构为「发碎片」页面

## 目标

将 `/blog/new` (NewPost.tsx) 从传统的博客文章编辑器改为**微信朋友圈/微博式的碎片发布界面**。

## 设计参考

| 参考 | 核心交互 |
|------|---------|
| 微信朋友圈 | 照片 + 文字，发布后以卡片时间线展示 |
| 微博 | 快速输入框 + 图片区 + 表情/话题 |
| Obsidian | 编辑区实时预览，支持 `$...$` 和 `$$...$$` LaTeX 数学公式 |

## 具体改动

### 1. NewPost.tsx — 完全重写

**页面布局**：

```
┌─────────────────────────────────────────┐
│  ← 返回        发碎片                   │  <- 顶部栏
├─────────────────────────────────────────┤
│                                         │
│  [添加照片区域]                          │  <- 可选的图片上传区
│                                         │
├─────────────────────────────────────────┤
│  ┌─────────────────────────────────┐    │
│  │  textarea 大输入框              │    │
│  │  支持 Markdown + LaTeX          │    │
│  │  $E = mc^2$                    │    │
│  │                                 │    │
│  └─────────────────────────────────┘    │
│                                         │
│  ┌─────────────────────────────────┐    │
│  │  实时预览区                      │    │
│  │  (Markdown + LaTeX 渲染)         │    │
│  └─────────────────────────────────┘    │
│                                         │
├─────────────────────────────────────────┤
│  [分类]  [标签]                         │  <- 底部元数据
└─────────────────────────────────────────┘
```

**关键改动**：

1. **顶部栏**：左边 `←` 返回，右边 `发碎片` 发布按钮（琥珀色圆角按钮），点击即发布跳转到生活碎片页
2. **图片区**：可选的图片添加区域（点击或拖拽上传），支持多图预览。暂时简化为单图 + 预览
3. **编辑区**：
   - 大 textarea，去掉之前所有 article 相关字段（title, slug, excerpt, readingTime 均自动生成）
   - 支持 Markdown 语法
   - 支持 `$...$` 行内 LaTeX 和 `$$...$$` 块级 LaTeX
4. **实时预览区**：编辑区下方，用 MarkdownRenderer + KaTeX 实时渲染，像 Obsidian 的 Live Preview
5. **底部元数据**：分类（emoji 选择器）+ 标签（简化，可选不填）
6. **去掉的字段**：title、slug、excerpt 均**自动生成**（title = 内容前30字，slug = 时间戳，excerpt = 内容前120字），用户无需手动填写

### 2. posts.ts — Post 类型扩展（可选）

`Post` 类型新增可选字段 `images?: string[]` 以支持多图，但这次先简化为单图（添加后插入 `![image](data:...)` 到正文）。

### 3. 发布流程

```
用户输入 → 点击「发碎片」→ 
  title = content 前30字
  slug = moment-{timestamp}
  excerpt = content 前120字
  readingTime = 自动计算
  → addPost() → 跳转到 /blog
```

## 不改的文件

- `posts.ts` — Post 类型保持不变
- `MarkdownRenderer.tsx` — 已支持 `remarkMath` + `rehypeKatex`
- `Blog.tsx`（生活碎片列表页）— 已经重构完成
- `BlogPost.tsx`（碎片详情页）— 保持不变
